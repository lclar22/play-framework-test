Restrict

Restrict requires users to have roles that match the ones defined in the constraint, in any combinaton of AND/OR/NOT

Restrict uses an ANDed set of roles to determine access to an action - these are defined as as Array[String[. For example, a user with the roles "foo" and "bar" could access a Restrictions-protected action that specified any of the following:

  • foo
  • bar
  • foo AND bar

However, a Restrict that required "foo", "bar" and "hurdy" would deny access to the user.

To define OR relationships, multiple groups within a list, i.e. List[Array[String]]

Specification
Result
Restrict(Array("foo")) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("foo", "bar")) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("foo", "!bar")) {
  Action {
    Ok(accessOk())
  }
}
Restrictions(Array("hurdy")) {
  Action {
    Ok(accessOk())
  }
}
